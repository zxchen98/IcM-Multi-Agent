Safety & Read/Write Matrix

What each server can and cannot do

ServerReadWriteConfirmation required before write?
ADO✅✅PR/branch/work-item writes — yes for merges/deletes
ES Build Data✅—n/a (read-only)
ICM✅—n/a (read-only — use portal API scripts for writes)
Kusto✅—n/a (read-only, management commands blocked)
Playwright✅✅No SSO/auth sites; destructive UI actions confirmed manually
EngineeringHub✅—n/a (read-only)
Work IQ Teams✅✅Yes — sends, deletes, member changes
Work IQ Mail✅✅Yes — send, reply, forward, delete
Azure MCP (optional)✅✅ (deploy tools)Yes — resource deploy/modify actions
Hard rules: never approve/merge PRs autonomously, never push to main/master, never delete branches without confirmation, never send Teams/emails without confirmation, never expose secrets.

Quick Decision Tree

Which MCP do I reach for?

Need to do something?
├─ ADO work items/PRs/pipelines/repos/wiki      → ADO MCP ✅
├─ Check build status/failures/ETA              → ES Build Data MCP ✅
├─ Raw KQL / RI calendar / build notes / crashes → Kusto MCP ✅
├─ Read ICM incident info                       → ICM MCP ✅
├─ Write/update ICM                              → ❌ portal API helper scripts
├─ Browse web page / scrape / BuildSVC UI       → Playwright MCP ✅
├─ Search eng.ms / TSGs / ServiceTree           → EngineeringHub MCP ✅
├─ Teams chats/channels/messages                → Work IQ Teams MCP ✅ (confirm sends)
├─ Read/search/send email                       → Work IQ Mail MCP ✅ (confirm sends)
├─ Azure resource mgmt / IaC / cost review      → Azure MCP (optional, tool_search first)
├─ Git push/commit                              → ❌ terminal git commands
└─ Approve/merge PR                             → ❌ human must do it

Full capability map: .github/instructions/Details/mcp-tools-index.detail.instructions.md • Server registry: .vscode/mcp.json
